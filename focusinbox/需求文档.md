# FocusInbox.io 需求文档

## 1. 产品定义

### 1.1 定位
极简社交媒体管理看板，专注于消息处理，避免算法推送的干扰。

### 1.2 目标用户
- 独立开发者
- 海外销售
- 推特大 V
- 寻找客户的自由职业者

### 1.3 核心价值
**无流量（Zero-Feed）**：进站只能看到消息，看不到算法推送的任何动态，帮助用户保持专注。

## 2. 核心功能

### 2.1 统一认证 (Unified Auth)
- 支持 Twitter (X) 和 LinkedIn 授权登录
- 简单快捷的 OAuth 认证流程
- 安全的 token 管理

### 2.2 双栏收件箱 (Dual-Column Inbox)
- **左纵栏**：展示最近的 X Mentions 和 DMs
  - 按时间顺序排列
  - 支持标记已读/未读
  - 支持快速回复

- **右纵栏**：展示最近的 LinkedIn Notifications 和 DMs
  - 按时间顺序排列
  - 支持标记已读/未读
  - 支持快速回复

### 2.3 快速回复片段 (Quick Reply Snippets)
- 允许用户预设 3-5 条常用回复
- 点击即填入回复框
- 支持编辑和管理常用回复
- 存储在 localStorage 中

### 2.4 防分心护盾 (Anti-Distraction Shield)
- 整个应用没有任何能通往 Twitter/LinkedIn 主页的链接
- 专注于消息处理，避免用户被其他内容分散注意力

## 3. 技术栈

### 3.1 框架
- Next.js 14 + Tailwind CSS（极致响应速度）

### 3.2 API 集成
- X (Twitter): 使用 twitter-api-v2 库
- LinkedIn: 使用 LinkedIn Marketing Developer Platform API

### 3.3 状态管理
- Zustand 或简单的 React Context（保持轻量）

### 3.4 持久化
- 用户的常用话术存放在 localStorage（MVP 阶段不需要数据库）

## 4. 界面设计

### 4.1 布局
- 响应式设计，适配桌面和移动设备
- 双栏布局（桌面端），单栏布局（移动端）
- 简洁的白色背景，减少视觉干扰

### 4.2 色彩方案
- 主色调：蓝色系（专业、信任）
- 辅助色：灰色系（中性、简洁）
- 强调色：绿色（成功）、红色（错误）

### 4.3 交互设计
- 平滑的过渡动画
- 清晰的视觉反馈
- 直观的操作流程
