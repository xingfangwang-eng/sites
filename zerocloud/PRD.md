# ZeroCloud.so 产品需求文档 (PRD)

## 1. 产品定义

### 1.1 定位
极简桌面客户端（Electron + Next.js），为用户提供 100% 离线的本地文档智能检索与问答能力。

### 1.2 目标用户
- 处理敏感合同的律师
- 拥有大量笔记的开发者
- 拒绝云端 AI 的隐私极客

### 1.3 核心卖点
**100% 离线**：用户选择一个本地文件夹，系统自动建立索引，提供一个 Chat 窗口进行智能问答。所有数据处理均在本地完成，无需联网，保护用户隐私。

## 2. 核心功能切片

### 2.1 Folder Picker（文件夹选择器）
**功能描述**：
- 一个醒目的主按钮，让用户选择本地目录
- 支持选择多个文件夹进行索引
- 显示已选文件夹列表，支持移除
- 记住用户上次选择的文件夹路径

**交互设计**：
```
┌─────────────────────────────────────┐
│  ZeroCloud.so                       │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  📁 选择文件夹              │   │
│  └─────────────────────────────┘   │
│                                     │
│  已索引文件夹：                      │
│  📂 /Users/lawyer/contracts        │
│  📂 /Users/dev/notes               │
│                                     │
└─────────────────────────────────────┘
```

### 2.2 Silent Indexer（静默索引器）
**功能描述**：
- 自动扫描目录下所有支持的文件类型
- 支持的文件格式：`.md`, `.txt`, `.pdf`
- 提取文本内容并进行本地向量化
- 使用 transformers.js 在本地运行，无需 GPU
- 显示索引进度和状态

**技术细节**：
- Embedding 模型：`Xenova/all-MiniLM-L6-v2`
- 向量维度：384 维
- 存储方式：本地 JSON 或 NeDB
- 增量更新：只处理新增或修改的文件

**索引流程**：
```
扫描文件夹 → 提取文本 → 文本分块 → 向量化 → 存储索引
```

### 2.3 The Killer Search（核心搜索功能）
**功能描述**：
- 一个类似 Spotlight 的搜索框
- 输入自然语言问题
- 基于本地向量数据库进行语义检索
- 调用本地 LLM（Ollama API）生成回答
- 显示引用来源（Source Attribution）

**交互设计**：
```
┌─────────────────────────────────────┐
│  ZeroCloud.so                       │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  🔍 搜索你的文档...          │   │
│  └─────────────────────────────┘   │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  用户：合同中的违约金条款是什么？     │
│                                     │
│  🤖 ZeroCloud：                     │
│  根据您的合同文档，违约金条款如下：   │
│                                     │
│  1. 违约金计算方式...               │
│                                     │
│  📄 来源：                          │
│  - contract_2024.md (第 15 行)      │
│  - terms_v2.txt (第 42 行)          │
│                                     │
└─────────────────────────────────────┘
```

**工作流程**：
```
用户输入问题 → 问题向量化 → 向量相似度检索 → 
获取相关文本块 → 构建 Prompt → 调用 Ollama → 
生成回答 → 显示引用来源
```

### 4.2 进程职责

#### Main Process（主进程）
- 文件系统访问（选择文件夹、读取文件）
- 文件监控（监听文件变化）
- 索引管理（文本提取、向量化）
- 向量数据库操作
- Ollama API 代理

#### Renderer Process（渲染进程）
- UI 渲染
- 用户交互处理
- 通过 IPC 与主进程通信
- 显示搜索结果和对话

## 5. 数据模型

### 5.1 文件索引 (FileIndex)
```typescript
interface FileIndex {
  id: string;                    // 唯一标识 (hash of path)
  path: string;                  // 绝对路径
  name: string;                  // 文件名
  extension: string;             // 扩展名
  size: number;                  // 文件大小 (bytes)
  modifiedTime: number;          // 最后修改时间
  indexedAt: number;             // 索引时间
  chunks: TextChunk[];           // 文本分块
}
```

### 5.2 文本分块 (TextChunk)
```typescript
interface TextChunk {
  id: string;                    // 唯一标识
  fileId: string;                // 所属文件 ID
  content: string;               // 文本内容
  startLine: number;             // 起始行号
  endLine: number;               // 结束行号
  embedding: number[];           // 384 维向量
}
```

### 5.3 搜索结果 (SearchResult)
```typescript
interface SearchResult {
  chunk: TextChunk;              // 匹配的文本块
  score: number;                 // 相似度分数 (0-1)
  file: FileIndex;               // 所属文件信息
}
```

### 5.4 对话消息 (ChatMessage)
```typescript
interface ChatMessage {
  id: string;                    // 唯一标识
  role: 'user' | 'assistant';    // 角色
  content: string;               // 消息内容
  sources?: SearchResult[];      // 引用来源
  timestamp: number;             // 时间戳
}
```

## 8. 性能指标

### 8.1 目标性能
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 启动时间 | < 3s | 应用冷启动 |
| 索引速度 | > 100 files/min | 普通文本文件 |
| 搜索响应 | < 500ms | 向量检索 + LLM 首 token |
| 内存占用 | < 500MB | 正常运行时 |

### 8.2 优化策略
- 使用 Worker 线程进行文件索引
- 向量数据库内存缓存
- 文本分块懒加载
- 索引增量更新

## 9. 安全与隐私

### 9.1 隐私承诺
- ✅ 所有数据处理均在本地完成
- ✅ 不连接任何云端服务（除用户自建的 Ollama）
- ✅ 不收集任何用户数据
- ✅ 索引数据本地加密存储

### 9.2 安全措施
- 文件路径脱敏（日志中）
- 安全的 IPC 通信
- 内容安全策略 (CSP)
